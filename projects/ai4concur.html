Finance staff processed hundreds of expense receipts monthly for SAP Concur. Each receipt required manual reading, category assignment, and field entry (date, amount, merchant, tax, currency). High volume, repetitive, error-prone.

Goal: Automate receipt → SAP Concur data entry end-to-end via event-driven pipeline.

Ideation — Scoped to Singapore Travel Receipts
tool: domain analysis · data: local transport + meal receipts (Singapore)

Scoped the POC to Singapore travel receipts (taxi, grab, meal) to bound the labeling effort. Singapore receipts share common format patterns — date top-right, merchant name top, total bottom — making them a tractable starting point before extending to hotel folios.

Label Studio — Document Classification Labels
tool: Label Studio · task: image classification labeling

Used Label Studio to label receipt images into 7 document classes, separating scanned physical receipts from digital/e-receipts and handling multi-receipt scans (multiple receipts on one page):

ClassMeaning
digital_needDigital e-receipt with required fields visible
digital_no_needDigital doc, no expense-relevant fields
receipt_singleSingle scanned physical receipt
receipt_22 receipts on one scanned page
receipt_33 receipts on one scanned page
receipt_multiple4+ receipts on one page
vaccinationVaccination cert (out-of-scope, exclude)
Key decision: Separating digital from scanned early avoids running OCR on already-structured digital PDFs unnecessarily.
Label Studio — Bounding Box Annotation for Receipt Segmentation
tool: Label Studio · task: object detection labeling

For scanned images containing multiple receipts on one page (receipt_2, receipt_3, receipt_multiple), used Label Studio's bounding box drawing tool to annotate individual receipt regions. These annotations trained the Faster RCNN model to segment multiple receipts from a single scan before OCR processing.

Model Training — CNN Classifier + Faster RCNN
framework: PyTorch · models: DenseNet → SqueezeNet (classifier), Faster RCNN ResNet50 (detector)

Trained two models in parallel:

  • Document classifier: Started with DenseNet; final model used SqueezeNet (lighter, faster inference, sufficient accuracy for 7-class task). Input 224×3px, normalized with ImageNet stats.
  • Receipt detector: Faster RCNN with ResNet50 backbone — detects individual receipt regions within a multi-receipt scan. Outputs bounding boxes per receipt.

Image augmentation used to expand the training pool: rotation (±15°), brightness/contrast variation, random crop — simulating real-world mobile photo conditions (angle, glare, partial crops).

Verified from codebase: Config confirms squeezenet1_0_Adam_Aug1 (SqueezeNet + Adam optimizer + augmentation round 1) and resnet50_070822 (ResNet50 Faster RCNN).
Pipeline Design — Image → OCR → Rules
tools: EasyOCR (custom weights) · regex NER · rule engine

The full processing pipeline:

  • Step 1 — Classify: CNN classifier routes image to digital or scanned path. Scanned → Faster RCNN crops individual receipt regions.
  • Step 2 — Rotate: Auto-rotation correction by running EasyOCR at 0°/90°/-90°/180° and selecting the angle with highest OCR confidence score.
  • Step 3 — OCR: Custom fine-tuned EasyOCR weights for English receipt text.
  • Step 4 — NER Extraction: Regex-based date extraction (handles 50+ date format permutations across separators and month formats). Amount extraction via regex + combinator (handles itemized subtotals summing to total).
  • Step 5 — Validator (7 flags): Receipt missing · amount >SGD30 threshold · amount mismatch · date mismatch · expired receipt (>90 days) · non-working day · policy timing (OT window check).
Key design: Receipts failing any flag route to a manual review queue — the system degrades gracefully rather than silently submitting wrong data.
SAP Concur API — Event-Driven Webhook Integration
tools: SAP Concur v4 REST API · FastAPI webhook · OAuth2

Original design processed receipts in batch (CSV export → run pipeline → upload). Redesigned to event-driven: SAP Concur fires a webhook on expense report submission; a FastAPI endpoint (/event_trigger) receives the event, extracts the report ID and user ID, fetches receipt images via SAP Concur v4 REST API (OAuth2 refresh token), and triggers the pipeline immediately.

This eliminated the batch delay and enabled near-real-time validation feedback to employees before the expense report went to a manager for approval.

Hotel Invoice Extension — Information Extraction Models
attempted: LayoutLM · Donut · Nougat · final: DETR + SetFit

Hotel folios are structurally different — multi-row itemized charges, table layouts, multi-page — requiring a document understanding model, not just OCR + regex.

Repeated the Label Studio annotation process (document-level bounding boxes for table regions) and evaluated three approaches:

ModelApproachOutcome
LayoutLMLayout-aware transformer for key-value extractionStruggled with wide multi-column hotel formats
DonutEnd-to-end document understanding (no OCR)Training converged but generalisation poor — hotel format too varied
NougatDocument parsing model for structured docsDid not converge on training data

Final approach: DETR (detr-doc-table-detection) for table region detection + SetFit for few-shot classification of extracted line items. POC overall accuracy settled at 70% — hotel folios remained the hard case due to wide format variability across hotel chains.

Root cause: Hotel invoice formats vary significantly across hotel chains — column widths, item naming, multi-currency rows. A rule-based approach would require per-vendor templates, which didn't scale.

⚡ Build-vs-Buy Post-Mortem

Finance chose a commercial SaaS for receipt processing after the POC. Tangible value: the POC defined the accuracy benchmarks used to evaluate SaaS vendors, and the vendor selection was informed directly by our field extraction tests. The technical work shaped the buy decision.
