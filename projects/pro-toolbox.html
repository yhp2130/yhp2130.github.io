$ cat problem.md

Process improvement consultants work across multiple business units, referencing SOPs, benchmarks, improvement reports, and standards. Finding relevant past work required manually searching through shared drives. Consultants were re-solving problems already documented elsewhere.

Goal: A RAG chatbot for cross-document synthesis — not just retrieval.

Delivery: Production-ready MVP delivered in 2 weeks using spec-driven development (GitHub Copilot SpecKit) — from concept to deployed application with API gateway configuration.

$ cat architecture.md — ExecutionPlan ReAct

Pro-Toolbox uses a plan-then-execute ReAct loop rather than step-by-step chain-of-thought. The planner produces a complete ExecutionPlan (list of SubGoal objects with dependency graph) before any tool is called — enabling parallel sub-goal execution via asyncio. Spans 4 knowledge sources: ARIS, Intranet, Confluence, and PRIMA.

  • Planner → LLM function-calling generates full ExecutionPlan with dependencies
  • ReactLoop → asyncio executes independent sub-goals in parallel; WorkingMemory tracks observations
  • SufficiencyEvaluator → custom LLM judge: enough context? yes → finalize / no → re-plan
  • CheckerAgent → post-generation citation gate before response returned
