$ cat architecture.md

Agent pattern: ReAct (Reason + Act)

A Planner agent receives the user query, reasons about which tools to call, and makes sequential tool calls. A Checker agent validates coherence before returning the response. On-prem LLM and embeddings — zero external API egress (data sovereignty).

$ cat key_decisions.md

  • Pure kNN retrieval (not hybrid) — ClaireGPT uses Elasticsearch dense_vector HNSW pure kNN. Sophistication moved up the stack to agent orchestration and synthesis, not retrieval blending.
  • CODEC used TF-IDF + SentenceBERT (in-memory) — original system had no Elasticsearch; all tensors were pickled in-memory. Elasticsearch was introduced only in Phase 3.
  • Semantic chunking — fixed-length split clauses mid-sentence, degrading LLM context quality. Semantic chunking was the single highest-impact change to answer quality.
  • On-prem LLM — CLM docs contain customer SLA terms and trade compliance data. Data sovereignty required: zero external API egress.
  • Planner + Checker — separates generation from validation. Checker reduced hallucination rate; claims must be traceable to retrieved source chunks.
  • BART as a tool — loose coupling. BART runs as an independent FastAPI service wrapped as a LangChain tool, deployable independently.
